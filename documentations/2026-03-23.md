## Subscription lifecycle

```
New user signup
  → profile row created
  → trigger: handle_new_profile inserts subscriptions(plan=free, status=active)

User clicks upgrade on pricing page
  → CheckoutButton → createCheckoutSession → Stripe Checkout
  → On success: Stripe fires checkout.session.completed
  → Webhook: upserts subscriptions row with plan + stripe IDs

User cancels / payment fails
  → Stripe fires customer.subscription.updated / deleted
  → Webhook: updates plan back to free or sets status=canceled
```

---

## Webhook events handled

| Event | Action |
|---|---|
| `checkout.session.completed` | Upsert subscription row with plan from price ID |
| `customer.subscription.updated` | Re-upsert (handles plan changes, renewals, past_due) |
| `customer.subscription.deleted` | Reset to free plan, clear stripe_subscription_id |

## Local webhook testing (step-by-step)

### 1. Install Stripe CLI

```bash
brew install stripe/stripe-cli/stripe
```

### 2. Login

```bash
stripe login
```

### 3. Forward webhooks to local dev server

```bash
stripe listen --forward-to localhost:3000/api/webhooks/stripe
```

The CLI prints a webhook signing secret like `whsec_...` — copy it into `.env.local` as `STRIPE_WEBHOOK_SECRET`.

### 4. Restart dev server so the new env var is picked up

```bash
npm run dev
```

### 5. Trigger test events

In a second terminal:
```bash
# Simulate a successful checkout
stripe trigger checkout.session.completed

# Simulate subscription update
stripe trigger customer.subscription.updated

# Simulate cancellation
stripe trigger customer.subscription.deleted
```

### 6. Create Stripe products and prices

In the Stripe dashboard (test mode):

1. Products → Add product → "Starter" → $19/month recurring → copy the **price ID** → paste into `STRIPE_PRICE_STARTER`
2. Products → Add product → "Pro" → $49/month recurring → copy the **price ID** → paste into `STRIPE_PRICE_PRO`

### 7. Run the DB migration

In Supabase SQL editor, run `supabase/migrations/004_subscriptions.sql`.
