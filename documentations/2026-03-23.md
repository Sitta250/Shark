## Subscription lifecycle

```
New user signup
  → profile row created
  → DB trigger: auto-inserts subscriptions(plan=free, status=active)

User upgrades
  → CheckoutButton → createCheckoutSession() → redirect to Stripe Checkout
  → Stripe fires: checkout.session.completed
  → Webhook: upserts subscription row (plan, status, stripe IDs, period end)

Renewal / plan change
  → Stripe fires: customer.subscription.updated
  → Webhook: re-upserts with latest status

Cancellation / payment failure
  → Stripe fires: customer.subscription.deleted
  → Webhook: resets plan to free, clears stripe_subscription_id
```

---

## Webhook events handled

| Event | Action |
|---|---|
| `checkout.session.completed` | Upsert subscription with plan mapped from price ID |
| `customer.subscription.updated` | Re-upsert (renewals, plan changes, past_due) |
| `customer.subscription.deleted` | Reset to free, clear subscription ID |

## What is still needed to fully complete Stripe

The code is done. Everything below is configuration and testing — no more code to write.

### 1. Run the DB migration

In Supabase SQL editor, run:
```
supabase/migrations/004_subscriptions.sql
```

This creates the subscriptions table, RLS policies, and the auto-create trigger for new users.

**Backfill existing users:** If there are already users in the database, run this after the migration to create their free subscription rows:
```sql
insert into subscriptions (user_id, plan, status)
select id, 'free', 'active' from profiles
on conflict (user_id) do nothing;
```

### 2. Create products in Stripe dashboard (test mode)

Go to **Stripe Dashboard → Products → Add product**:

| Product | Price | Billing | Env var to fill |
|---------|-------|---------|-----------------|
| Starter | $19   | Monthly recurring | `STRIPE_PRICE_STARTER` |
| Pro     | $49   | Monthly recurring | `STRIPE_PRICE_PRO` |

After creating each, copy the **price ID** (starts with `price_`) and paste it into `.env.local`.

---

### 3. Configure the Customer Portal

In **Stripe Dashboard → Settings → Customer portal**:

- Enable: Cancel subscriptions
- Enable: Update payment method
- Set the **Return URL** to: `https://yourdomain.com/dashboard` (or `http://localhost:3000/dashboard` for local)
- Save

Without this, `createPortalSession()` will throw an error when users try to manage billing.

---

### 4. Register the production webhook endpoint

When deploying to Vercel:

1. Stripe Dashboard → **Developers → Webhooks → Add endpoint**
2. Endpoint URL: `https://yourdomain.com/api/webhooks/stripe`
3. Select events:
   - `checkout.session.completed`
   - `customer.subscription.updated`
   - `customer.subscription.deleted`
4. Copy the **signing secret** (`whsec_...`) and add it to Vercel environment variables as `STRIPE_WEBHOOK_SECRET`

Note: The local `stripe listen` secret and the production Stripe dashboard secret are **different values**. Keep them separate per environment.
