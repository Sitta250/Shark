## 5. AI report generation pipeline

Built full intake → OpenAI → typed JSON → Supabase pipeline.

**Schema (`lib/report/schema.ts`):**
9 typed sections:
- `executive_verdict` — score (0-100), recommendation (go/caution/avoid), headline, summary, key strengths
- `customer_and_pain` — primary customer, pain intensity, willingness to pay, evidence
- `market_attractiveness` — size estimate, growth trend, tailwinds, headwinds
- `competitors` — landscape summary, competitor cards (name/description/weakness), positioning gap
- `business_model` — recommended model, pricing strategy, revenue streams, key assumptions
- `mvp_features` — core/phase2/avoid features, build time estimate
- `go_to_market` — launch channel, first 100 customers, content angle, partnerships
- `risks` — array of (title, description, severity, mitigation)
- `next_steps` — week_1[], month_1[], quarter_1[]

**Prompt (`lib/report/prompt.ts`):**
- `buildSystemPrompt()` — strict JSON schema contract, quality rules (specific not generic, 3-5 competitors, 4-6 risks)
- `buildUserPrompt(intake)` — maps all intake fields to readable context

**OpenAI call (`lib/report/generate.ts`):**
- Model: `gpt-4o-mini`, temperature: 0.4
- `response_format: { type: "json_object" }`

**Server actions (`lib/report/actions.ts`):**
- `getReport(projectId)` — fetches latest report row
- `generateReport(projectId)` — verifies ownership → checks intake complete → inserts pending row → calls OpenAI → updates completed/failed

**Components:**
- `components/report/section-card.tsx` — shared `SectionCard`, `BulletList`, `LabeledText`
- `components/report/report-view.tsx` — 9 section components with badge styling for verdict/pain/growth/risk
- `components/report/generate-report-button.tsx` — `useTransition` loading state, ~20-30s hint, `router.refresh()` on success

## 6. Workspace modules

Built full workspace layer: tasks, milestones, assumptions, notes.

**Server actions (`lib/workspace/actions.ts`):**
- Full CRUD for all 4 entities (create/update/delete + fetch)
- `seedWorkspaceFromReport(projectId, sections)` — bulk seeds workspace from report:
  - `week_1` → tasks (status: todo)
  - `month_1` + `quarter_1` → milestones (status: planned)
  - `risks` → assumptions (status: untested)

**Components:**
- `workspace-shell.tsx` — tab bar (Report / Tasks / Milestones / Assumptions / Notes) via URL `?tab=` param, active underline in secondary color
- `tasks-module.tsx` — grouped by status, circle toggle to cycle status, hover-reveal delete, inline add form
- `milestones-module.tsx` — sorted by date, click badge to cycle planned→achieved→missed, date input on add
- `assumptions-module.tsx` — expandable cards with notes textarea (saves on blur), dropdown status, inline add
- `notes-module.tsx` — 2-col card grid, click to edit inline, newest first
- `seed-banner.tsx` — shown when workspace empty, one-click seed from report sections

**Updated `app/(app)/projects/[id]/page.tsx`:**
- Fetches all workspace data server-side in parallel
- 3-state routing: intake incomplete → intake flow | report not generated → report prompt | report done → tabbed workspace
- Tab content rendered server-side; shell handles client-side tab links
- `?tab=` search param drives active tab (defaults to `report`)

**Database (`supabase/migrations/003_notes.sql`):**
- New `notes` table with RLS policies (select/insert/update/delete scoped through project ownership)
- All other workspace tables (tasks, milestones, assumptions) were already in `001_initial_schema.sql`
